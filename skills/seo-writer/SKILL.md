# SEO Writer

用于批量生产SEO文章，兼顾搜索排名和真人阅读体验。设计为多 agent 共享——每个 agent 在「品牌定制层」注入自己的行业知识和品牌调性，避免输出同质化。

## 核心原则（三层必须全部通过）

### 第一层：反 AI 模式（human-writing 规则）

这是底线。任何违反此层规则的文章，不允许输出。

**词汇禁区（出现即替换）：**

| 禁用词 | 替换建议 |
|--------|----------|
| 本质上 | 说到底 / 其实 / 说白了 |
| 意味着 | 所以 / 那结果呢 |
| 至关重要的 | 关键的 / 重要的 |
| 令人惊叹的 | 让人印象深的 / 挺好用的 |
| 显著的 / 显著的提升 | 明显更好 / 涨了很多 |
| 不可或缺 | 必备的 / 少了它不行 |
| 赋能 | 帮助 / 让…能 |
| 打造 | 做出来 / 做出 / 做出 |
| 构筑 | 建立 / 形成 |
| 赋能 / 驱动 | 推动 / 让…更强 |
| 全面提升 | 整体变好 / 提升不少 |
| 深度 | 有多想 / 多透彻 |
| 精细 | 细致 / 具体 |
| 革新 | 改变 / 换新 |
| 卓越 | 很棒 / 出色 |
| 旨在 | 为了 / 目的是 |
| 助力 | 帮助 |
| 有效提升 | 真的变好了 / 涨了 |
| 核心优势 | 主要好处 |
| 全面覆盖 | 都能包含 / 全都覆盖到 |

**句式禁区：**

- ❌ "首先...其次...最后" → 替换为自然转场或数字列表
- ❌ "总而言之" / "综上所述" → 替换为具体的回扣句
- ❌ "值得注意的是" / "不难发现" → 删掉，直接说
- ❌ "让我们来看" / "接下来让我们" → 删掉
- ❌ "在当今...时代" / "随着...发展" → 直接切入具体场景
- ❌ "这意味着什么？" → 换成"所以呢？"或者直接删掉
- ❌ "说白了" → 换成"其实就是"
- ❌ "换句话说" → 换成"你想想看"或者直接说
- ❌ "不可否认" → 删掉，直接陈述观点
- ❌ "深入..." → 换成"仔细看..."或者直接描述动作

**标点禁区：**

- 不用冒号「：」→ 用逗号替代
- 不用破折号「——」→ 用逗号或句号替代
- 不用双引号「""」或「"」→ 用「」替代，或直接不加引号

**结构禁区：**

- 不用 markdown 多级标题体系（一级标题二级标题那种）
- 段落之间不要用粗体切割（每篇最多两处加粗）
- 不用 bullet point 列表（超过3个要改成散文叙述）

---

### 第二层：SEO 规则（文章必须满足）

**关键词布局：**

- 标题必须包含主关键词（第一位出现）
- 首段前100字内必须出现主关键词
- 全文关键词密度：1.5%–2.5%（不要堆砌，自然出现）
- 长尾关键词分布在各段落的子标题或首句中

**结构要求：**

- 字数：1500-3000字（低于1500字排名竞争力弱，高于3000字容易冗余）
- 段落要短（每段平均3-5句，避免超过8句的超长段落）
- H2/H3 小标题用口语化转场句代替（"说完了A，这块说说B"而不是"二、B的特点"）
- 开头必须从具体场景切入（一个事件、一个数字、一个问题），禁止宏大叙事开头
- 结尾必须有具体的CTA或行动指引（"如果你想知道更多，可以…"）

**EEAT 信号（Google评价内容质量的框架）：**

- E（Experience经验）：文章里要有"我试过"、"我用过"、"我看过"这类第一人称体验描述，哪怕是你帮用户做的调研，也要说清楚"我帮你们测了X个产品，发现…"
- E（Expertise专业）：专业内容要有具体数据、来源、案例支撑，不能只说"效果很好"
- A（Authoritative权威）：引用来源要具体，"XX媒体测试了"比"行业报告显示"更可信
- T（Trustworthy可信）：不能有明显夸大，说"最好的"改成"我们用过最顺手的"，说"第一"改成"我们测试下来排在前列的"

### 第三层：品牌定制（每个 agent 必须自定义）

在动笔之前，先填写以下品牌信息卡（每次任务都要更新，不是写一次就完）：

```
【品牌信息卡】

品牌名称：
产品/服务：
目标读者是谁？（具体描述，不是"需要装修的人"而是"买了二手学区房、预算15万、第一次装修不知道从哪下手的年轻夫妻"）
读者最关心什么？（3个以内，具体）
读者已经在用什么方案/竞品？（他们的现状是什么？）
他们踩过什么坑？
我们的产品在哪个点上是他们最在意的？
文章目的是什么？（引流/转化/品牌曝光/回答用户问题？）
主推关键词：（1个主词+2-3个长尾词）
文章角度：（从哪个具体切入点走？禁止"全方位介绍"）
字数要求：（1500/2000/2500/3000）
发布平台：（官网/公众号/知乎/头条/百家号）
```

## 写作流程

### 第一步：填品牌信息卡（5分钟）

不填完不开始写。没有例外。

### 第二步：确定文章角度（5分钟）

一个好的SEO文章角度必须同时满足：
- **搜索意图匹配**：用户搜这个词的时候是想干什么？
- **差异化**：你从这个角度写，跟别人有什么不同？
- **内容可执行**：读者读完能真的做点什么，不能只是"知道了"

如果品牌信息卡里的"文章角度"已经有明确方向，跳过这一步。

如果角度模糊，跟用户确认："你们想从哪个方向切入？我列出三个选项你们选一个。"

### 第三步：列大纲（10分钟）

大纲不需要漂亮，但必须回答这三个问题：

1. 读者读完这篇文章，脑子里会多知道哪3件事？
2. 文章开头用什么钩子？（具体场景/数字/问题/反常识事实）
3. 文章结尾的落点是什么？（一个行动/一个结论/一个思考）

大纲模板（不要太详细，关键词+段落大意即可）：

```
开头钩子：XXXX
第一段：（主题+读者痛点）
第二段：（核心内容展开）
第三段：（案例/数据支撑）
第四段：（实际怎么做的步骤或建议）
第五段：（升华或回扣开头）
结尾CTA：XXXX
```

### 第四步：写作（按字数要求分配时间）

初稿不用追求完美，先写完。写的时候注意：

- 每段开头用短句，不用长句引入
- 关键观点后面跟一个具体案例或数据
- 段落末尾留一句扣主线的话（防止偏太远）
- 口语化优先，能用"你"就不用"用户"
- 主动承认缺点和局限（"这个东西也有不足，XX方面确实不如YY"）——真实感来自不完美

### 第五步：三层自检

**自检L1 — 反AI模式扫描：**

- 全文搜索禁用词表格，出现即替换
- 全文搜索句式禁区，有即改
- 搜索"。"连续出现次数（超过连续3个句号说明句子太短）
- 确认全文没有超过5处的加粗
- 确认没有超过3处的 bullet/numbered list

**自检L2 — SEO合规扫描：**

- 主关键词出现在标题中 ✓/✗
- 首段100字内有关键词 ✓/✗
- 全文字数在要求范围内 ✓/✗
- 每个长尾词至少出现1次 ✓/✗
- 结尾有CTA ✓/✗
- 没有死链承诺或夸大宣传 ✓/✗

**自检L3 — 品牌一致性扫描：**

- 调性是否符合品牌信息卡中的描述？✓/✗
- 读者画像准确吗？（读者会不会觉得"说的就是我"？）✓/✗
- 文章角度是否和品牌想传递的核心信息一致？✓/✗
- 有没有人味细节？（具体数字、具体场景、真实案例）✓/✗

三层全部通过才能输出。任何一层不通过，打回修改。

## 输出格式

文章输出格式：

```
【文章标题 - 包含主关键词】

【正文】

---
本文由[品牌名]原创发布。
如需转载或引用，请联系[联系方式]。
```

最后一行不要加"作者：XXX"，SEO文章不需要个人IP感。
